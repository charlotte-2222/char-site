Fembot & It's Features

In the Fall of 2019, a friend and I began co-writing a Python bot "Gizmo", which served the primary purpose of being a tool for our D&D group, featuring tools such as webscraping with BeautifulSoup. Gizmo served as my introduction to Discord.py, and Python in general.

Fembot officially began in February 2020, and now hosts several features of which I'm incredibly proud. For instance, recently in 2021 several major Discord bots were hit with C&D's by YouTube for their (the Bot dev's) usage of the YT-API to stream music to voice channels. Fortunately for me, I have not received one of these.

I currently use the YoutubeDL library to search and stream URL's from YouTube. Instead of outright re-streaming to voicechannel, I've opted to download the audio to directory and then stream. The bot will automatically remove the downloaded audio once it exits the voice channel (Either by being told to leave, or by timing out due to inactivity)

Fembot is not purely a music bot, of course. It is general purpose. Using the DiscordAPI, I've been able to create several Admin/Server setup commands including commands to mute, ban, and warn members. My approach is to create a log channel for all admin actions, with each of these commands there is an optional argument to provide a reason for the admin action. This argument is taken from an array and converted to a string - then displayed back in a rich embed format along with user information, all in the logging channel.
